## 폴리곤 확보 방법 (3가지 옵션)

### 옵션 1: 아실 데이터 캡처 (권장)
**장점**: 모든 사업유형 통합, 폴리곤 포함, 진행단계 포함
**단점**: 수동 캡처 필요 (서울 전역 지도 이동하며 캡처)
**방법**:
```javascript
// 아실 콘솔에서 실행
fetch('/json/data_redevelop.jsp')
  .then(r => r.json())
  .then(d => {
    console.log('건수:', d.length);
    copy(JSON.stringify(d));
  });
```
**예상 결과**: 서울시 전체 정비사업 데이터 (수백~천 건)

### 옵션 2: 주소 기반 지오코딩
**장점**: 공식 데이터 사용
**단점**: 폴리곤 아닌 점(Point) 데이터만 가능
**방법**: 클린업 주소 → 카카오/네이버 지오코딩 API → 중심점 좌표

### 옵션 3: 도시계획 고시문에서 폴리곤 추출
**장점**: 정확한 공식 폴리곤
**단점**: 수백 개 고시문 수동 확인 필요, 비현실적

## 권장 방안

### 1단계: 아실 전체 데이터 캡처
아실은 이미 모든 정비사업 데이터를 통합해서 보유하고 있음.
서울시 전역을 캡처하면 모아타운, 가로주택, 신통기획 등 모두 포함.

### 2단계: 데이터 정제 및 DB 적재
- GeoJSON 변환
- PostgreSQL/PostGIS 적재
- 진행단계별 필터링

### 3단계: 뷰어 구축
- 카카오맵/Leaflet 기반 뷰어
- 사업유형별, 진행단계별 필터
- 아실과 동일한 UX 구현

## 아실 전체 캡처 방법

아실 지도에서 데이터는 화면에 보이는 영역만 로드됨.
전체 서울 데이터를 얻으려면:

1. 아실 지도를 서울 전체가 보이도록 축소
2. 또는 각 구별로 이동하며 여러 번 캡처 후 병합

### 더 나은 방법: 아실 API 분석
아실 네트워크 탭에서 API 호출 패턴을 분석하면,
bbox 파라미터로 영역을 지정해서 전체 데이터를 가져올 수 있을 수 있음.
